Lab Overview & Business Impact

The Challenge: Information Overload üò©

In today's fast-paced environment, users are often confronted with vast amounts of text. Quickly understanding the essence of long documents like Salesforce Knowledge Articles, detailed Case Descriptions, or comprehensive Product Manuals is crucial but time-consuming.

The Solution: Knowledge Companion ‚ú®

This lab guides you in building a "Knowledge Companion" tool. Powered by the Gemini API (e.g., Gemini Flash 1.5 or newer compatible Flash models like a future Flash 2.0 if available), this tool integrates directly into Salesforce to provide instant summaries of lengthy text content. This significantly enhances productivity by allowing users to grasp key information at a glance.

Real-Life Use Cases üí°

  • Support Agents: Quickly summarize complex case histories or long knowledge articles to resolve issues faster.
  • Sales Teams: Get rapid summaries of account details, past communications, or opportunity notes.
  • Product Managers: Condense user feedback or technical documentation.
  • Any Salesforce User: Efficiently process information from lengthy text fields across various objects.

Lab Goal üéØ

By the end of this lab, you will have built a solution that can:

  • Summarize content from a rich text area field on a Salesforce Knowledge Article using the Gemini API.
  • Trigger this summarization process via a custom Lightning Web Component (LWC) button.
  • Display the AI-generated summary clearly within a modal dialog on the screen.
  • Explore your creativity in crafting effective prompts to the Gemini API to summarize different types of content from a Knowledge Article field in innovative ways.

This hands-on experience will cover Salesforce configuration, Apex development, and LWC creation.

For Lab Simplicity: We are putting the API key directly into the Apex code and constructing the endpoint URL with it. This simplifies setup for the lab.

‚ö†Ô∏è Security Best Practice (for real projects): DO NOT hardcode API keys or include them directly in endpoint URLs in your code.

Lab Step 3: Apex Code Walkthrough üë®‚Äçüíª

This Apex class (GeminiAPIService.cls) makes the callout to the Gemini API using the latest Flash model (e.g., Gemini Flash 1.5 or newer compatible versions like Flash 2.0, if available and supported by the public API) and the provided API key.

Action: Create this Apex class in your Salesforce org.


// File: GeminiAPIService.cls
public with sharing class GeminiAPIService {

    private static String getProvidedApiKey() {
        // TODO: LAB PARTICIPANT - REPLACE 'KEY_PROVIDED_BY_INSTRUCTOR' WITH THE ACTUAL API KEY.
        String apiKey = 'KEY_PROVIDED_BY_INSTRUCTOR'; 

        if (String.isBlank(apiKey) || apiKey == 'KEY_PROVIDED_BY_INSTRUCTOR') {
            System.debug(LoggingLevel.ERROR, 'GeminiAPIService: API Key is missing or is the placeholder. Update with the provided key.');
        }
        return apiKey;
    }

    @AuraEnabled(cacheable=false) 
    public static String getPageSummaryFromGemini(String articleFieldContent) {
        if (String.isBlank(articleFieldContent)) {
            return 'Error: Knowledge Article field content is blank.';
        }
        
        String GEMINI_API_KEY = getProvidedApiKey();
        if (String.isBlank(GEMINI_API_KEY) || GEMINI_API_KEY == 'KEY_PROVIDED_BY_INSTRUCTOR') {
            return 'Error: The provided Gemini API Key is not configured in Apex.';
        }

        // Using latest available Flash model. Replace 'gemini-1.5-flash-latest' 
        // with a specific "Flash 2.0" model identifier if provided and available.
        String modelIdentifier = 'gemini-1.5-flash-latest'; 
        String endpoint = 'https://generativelanguage.googleapis.com/v1beta/models/' + modelIdentifier + ':generateContent?key=' + GEMINI_API_KEY;

        // Updated prompt for summarizing a Knowledge Article field
        String prompt = 'You are an expert AI assistant. Your task is to summarize the content of a Salesforce Knowledge Article field. ' +
            'The content might be technical, procedural, or informational. ' +
            'Provide a concise, well-structured summary in HTML format. ' +
            'Use bullet points (<ul><li>...</li></ul>) for lists or key takeaways where appropriate. ' +
            'Focus on extracting the core information and main points from the provided text. ' +
            'The summary should be easily readable and directly usable by someone looking for the essence of the article content. ' +
            'Input text from Knowledge Article field:\n\n' + articleFieldContent;

        Map<String, Object> requestBody = new Map<String, Object>{
            'contents' => new List<Object>{
                new Map<String, Object>{
                    'parts' => new List<Object>{
                        new Map<String, Object>{ 'text' => prompt }
                    }
                }
            }
            // Optional: 'generationConfig': new Map { 'maxOutputTokens' => 200 }
        };

        HttpRequest req = new HttpRequest();
        req.setEndpoint(endpoint); 
        req.setMethod('POST');
        req.setHeader('Content-Type', 'application/json; charset=utf-8');
        req.setTimeout(60000); 
        req.setBody(JSON.serialize(requestBody));

        Http http = new Http();
        String summaryResult = '';

        try {
            HttpResponse res = http.send(req);
            System.debug('Gemini API Callout Status: ' + res.getStatusCode() + ', Body Len: ' + (res.getBody() != null ? res.getBody().length() : 0));

            if (res.getStatusCode() == 200) {
                Map<String, Object> jsonResponse = (Map<String, Object>) JSON.deserializeUntyped(res.getBody());
                List<Object> candidates = (List<Object>) jsonResponse.get('candidates');
                if (candidates != null && !candidates.isEmpty()) {
                    Map<String, Object> firstCandidate = (Map<String, Object>) candidates[0];
                    Map<String, Object> content = (Map<String, Object>) firstCandidate.get('content');
                    if (content != null) {
                        List<Object> parts = (List<Object>) content.get('parts');
                        if (parts != null && !parts.isEmpty()) {
                            Map<String, Object> part = (Map<String, Object>) parts[0];
                            summaryResult = (String) part.get('text');
                        } else { summaryResult = 'Error: No "parts" in API response.'; }
                    } else { summaryResult = 'Error: No "content" in API candidate.'; }
                } else { 
                    summaryResult = 'Error: No "candidates" in API response.';
                    if (jsonResponse.containsKey('error')) {
                        Map<String, Object> errorDetails = (Map<String, Object>)jsonResponse.get('error');
                        summaryResult += ' API Error: ' + String.valueOf(errorDetails.get('message'));
                    }
                }
            } else {
                summaryResult = 'Error from Gemini API: Status ' + res.getStatusCode() + '. Details: ' + res.getBody();
            }
        } catch (Exception ex) { // Catch generic Exception for broader error capture
            summaryResult = 'Error during API callout: ' + ex.getMessage();
            System.debug(LoggingLevel.ERROR, 'GeminiAPIService Exception: ' + ex.getMessage() + '\\n' + ex.getStackTraceString());
        }
        return summaryResult;
    }
}

2. knowledgeCompanion.js


// File: knowledgeCompanion.js
import { LightningElement, api, wire, track } from 'lwc';
import { getRecord, getFieldValue } from 'lightning/uiRecordApi';
import getPageSummaryFromGemini from '@salesforce/apex/GeminiAPIService.getPageSummaryFromGemini';

// --- IMPORTANT: Field API Name to Summarize ---
// TODO: LAB PARTICIPANT - Update this import to point to the API name of the 
// rich text field on YOUR Knowledge__kav object that you want to summarize.
// Example: import KNOWLEDGE_CONTENT_FIELD from '@salesforce/schema/Knowledge__kav.Your_Custom_Field_API_Name__c';
import KNOWLEDGE_CONTENT_FIELD from '@salesforce/schema/Knowledge__kav.Content_for_Summarization__c'; 
// Defaulting to 'Content_for_Summarization__c', change if your field is different.

export default class KnowledgeCompanion extends LightningElement {
    @api recordId; // Automatically gets the ID of the record page

    @track summaryText = '';
    @track isLoading = false;
    @track error;
    @track isModalOpen = false;
    _wiredArticleData;

    // Wire service to fetch the content of the specified field
    @wire(getRecord, { recordId: '$recordId', fields: [KNOWLEDGE_CONTENT_FIELD] })
    wiredArticle({ error, data }) {
        if (data) {
            this._wiredArticleData = data;
            this.error = undefined;
        } else if (error) {
            this.error = 'Failed to load article content. Check LWC field API name & record access.';
            console.error('LWC @wire(getRecord) error:', JSON.stringify(error));
            this._wiredArticleData = undefined;
        }
    }

    // Getter to extract the actual field value from the wired data
    get articleContent() {
        if (this._wiredArticleData) {
            return getFieldValue(this._wiredArticleData, KNOWLEDGE_CONTENT_FIELD);
        }
        return undefined;
    }

    // Handles the click event of the "Summarise Article Content" button
    async handleSummarizeArticleField() {
        this.isLoading = true;
        this.error = undefined;
        this.summaryText = '';

        const contentValue = this.articleContent;

        if (!contentValue || String(contentValue).trim() === '') {
            this.error = \`The field '\${KNOWLEDGE_CONTENT_FIELD.fieldApiName}' configured in the LWC is empty or contains no text to summarize on this record.\`;
            this.isLoading = false;
            return;
        }
        
        // Remove HTML tags from rich text content before sending to API
        let plainTextContent = this.stripHtml(String(contentValue));

        if (!plainTextContent || plainTextContent.trim() === '') {
            this.error = 'The content of the field became empty after attempting to strip HTML tags. Original content might have only contained HTML structures or was whitespace.';
            this.isLoading = false;
            return;
        }

        try {
            // Call the Apex method, passing the plain text content
            const result = await getPageSummaryFromGemini({ articleFieldContent: plainTextContent }); // Match Apex param name
            
            if (result) {
                if (result.toLowerCase().startsWith('error:')) {
                    this.error = result; // Display error from Apex
                } else {
                    this.summaryText = result; // Store successful summary (expected to be HTML)
                    this.isModalOpen = true;   // Open the modal
                }
            } else {
                this.error = 'Received an empty or null response from the summarization service.';
            }
        } catch (e) {
            this.error = 'An error occurred while trying to get the summary: ' + (e.body?.message || e.message);
            console.error('LWC Error calling Apex getPageSummaryFromGemini:', JSON.stringify(e));
        } finally {
            this.isLoading = false; // Hide spinner
        }
    }
    
    // Utility function to strip HTML tags from a string
    stripHtml(html) {
        if (!html) return "";
        try {
            let doc = new DOMParser().parseFromString(html, 'text/html');
            return doc.body.textContent || "";
        } catch (e) {
            // Basic fallback, less reliable for complex HTML
            return html.replace(/<[^>]*>?/gm, '');
        }
    }

    // Handles closing the summary modal
    closeModal() {
        this.isModalOpen = false;
        this.summaryText = ''; // Clear summary when modal is closed
    }
}

Alternative Approach: Leveraging Salesforce Native AI ‚òÅÔ∏è

While this lab focuses on direct integration with an external LLM like Gemini for learning purposes, it's important to be aware of Salesforce's evolving native AI capabilities.

Using Einstein and Platform AI Features

Salesforce is increasingly embedding Generative AI into its platform (e.g., through Einstein GPT, Service Cloud AI, Sales Cloud AI, and features similar to "Agent Assist" or "Copilot" experiences). For tasks like summarization, especially within standard objects like Knowledge or Case, Salesforce may offer out-of-the-box or configurable solutions.

Conceptual Steps for a Native Salesforce AI Summarization:

  1. Identify Use Case & Object: Determine which object (e.g., Knowledge Article - Knowledge__kav) and which specific field (e.g., a rich text field like Article_Body__c) requires summarization.
  2. Explore Salesforce AI Services:
    • Check for existing Einstein features for that object (e.g., Einstein for Service might have Case Summarization, Einstein Article Recommendations for Knowledge).
    • Look into features like Prompt Builder or specific "Copilot" actions that allow you to invoke LLMs managed by Salesforce.
  3. Configure Prompt Templates (if applicable):
    • If using a feature like Prompt Builder, you would create a prompt template tailored for summarizing Knowledge Article content. This template guides the Salesforce-managed LLM.
    • Example Prompt Focus: "Summarize this Knowledge Article field content into 3 key bullet points suitable for an support agent."
  4. Trigger Summarization:
    • This might be an out-of-the-box button provided by the Salesforce feature.
    • It could be invoked via a Flow that calls a Prompt Template or an Invocable Apex action that interacts with the Salesforce AI services.
    • A custom LWC could also trigger such a Flow or Apex action.
  5. Display Summary:
    • The native feature might have its own UI for displaying summaries.
    • Summaries could be displayed in standard components, custom LWCs, or even written back to a field (though real-time display is often preferred).

Benefits of Using Native Salesforce AI:

  • Data Residency & Trust: Data typically stays within the Salesforce Trust Boundary.
  • Contextual Awareness: Salesforce-managed LLMs can be more easily grounded with your specific org data and context.
  • Simplified Integration: Often less custom code is needed compared to integrating with an external API. Configuration might be more prominent.
  • Managed Service: Salesforce handles the underlying LLM infrastructure.

As you continue your Salesforce development journey, keep an eye on new releases and features related to Generative AI on the Salesforce platform, as they can offer powerful and integrated solutions for use cases like document summarization.

A. How to Obtain Your Own Google Gemini API Key

  1. Go to Google AI Studio.
  2. Sign in with your Google account.
  3. You might need to enable the "Generative Language API" in a Google Cloud Project. Google AI Studio often guides you through this. Ensure billing is enabled for that project if required by Google Cloud for API usage beyond free tiers.
  4. In Google AI Studio, click "Get API key" then "Create API key".
  5. Copy the generated API key and store it securely.

B. Using Named Credentials in Salesforce (Best Practice for API Integration)

Named Credentials securely manage authentication and endpoint details for callouts.

  1. Store API Key Securely (Example): Use a Protected Custom Setting (e.g., `API_Keys__c` with a field `Gemini_Key__c`).
  2. Create Named Credential:
    • Setup > Named Credentials > New.
    • **Label:** e.g., `My_Secure_Gemini_API`
    • **Name:** (Auto-filled)
    • **URL:** `https://generativelanguage.googleapis.com`
    • **Identity Type:** `Anonymous` (often suitable if key is in header)
    • **Authentication Protocol:** `No Authentication`
    • **Custom Headers:**
      • Header Name: `x-goog-api-key`
      • Header Value: `{!'YOUR_API_KEY_HERE'}` (You can directly paste the key for testing, or for production, use a formula like `{!$Setup.API_Keys__c.Gemini_Key__c}` if your key is in a Custom Setting. Note: Direct merge fields for Custom Settings in Named Credential headers can be tricky; often better to set header in Apex after retrieving from Custom Setting.) A common pattern is to leave API Key out of Named Credential custom headers and add it in Apex if it's dynamic or from secure storage.
    • Save.
  3. Update Apex to Use Named Credential:
