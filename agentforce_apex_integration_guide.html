Agentforce with Apex Integration

This guide demonstrates how to use Apex to create custom actions for your Agentforce agent. We will create an Apex class that allows an agent to create a follow-up task for a case and send a confirmation email to the customer.

Overall Process:

  • Step 1: Create the Apex class.
  • Step 2: Add extra instructions to your existing topic.
  • Step 3: Create the agent custom action based on the Apex class.
  • Step 4: Add the new action to your agent's topic.
  • Step 5: Try out the new functionality.

Step 1: Create the Apex Class Initial Setup

First, we'll create the Apex class that defines the custom logic.

1.1 Launch Code Builder & Create Class

  • From App Launcher, open Code Builder and click Launch.
  • Once loaded, open the command palette (CMD + SHIFT + P on Mac or CTRL + SHIFT + P on PC).
  • Search for Apex and click Create Apex Class.
  • Enter CaseFollowUpAgentAction as the class name and accept the default directory.

Step 1: Create the Apex Class 1.2 Apex Code - Structure

Replace the default Apex class content with the following code.

public with sharing class CaseFollowUpAgentAction { @InvocableMethod(label='Create Follow-Up Task & Email' description='Creates a customizable follow-up task and sends a plain text email to the customer.') public static List createFollowUpTaskAndEmail(List requests) { List responses = new List(); FollowUpResponse response = new FollowUpResponse(); // Validate input request. if (requests == null || requests.isEmpty() || requests[0].caseNumber == null) { response.isSuccess = false; response.outcomeMessage = 'Error: Case Number not provided in the request.'; responses.add(response); return responses; } FollowUpRequest request = requests[0]; String caseNumber = request.caseNumber; try { Case caseRecord = [ SELECT Id, CaseNumber, ContactId, Contact.Email, Contact.FirstName, Contact.LastName FROM Case WHERE CaseNumber = :caseNumber LIMIT 1 ]; if (caseRecord.Id == null) { throw new HandledException('Case with CaseNumber ' + caseNumber + ' not found.'); } if (caseRecord.ContactId == null || String.isBlank(caseRecord.Contact.Email)) { throw new HandledException('Case contact or contact email is missing for Case : ' + caseNumber); } // --- 1. Create the Follow-Up Task --- Task followUpTask = new Task(); followUpTask.Subject = 'Follow-Up for Case: ' + caseRecord.CaseNumber; followUpTask.WhatId = caseRecord.Id; followUpTask.WhoId = caseRecord.ContactId; followUpTask.OwnerId = UserInfo.getUserId(); followUpTask.Status = 'Open'; followUpTask.Priority = 'High'; insert followUpTask; System.debug('Successfully inserted Task: ' + followUpTask.Id); // --- 2. Create and Send the Confirmation Email using SingleEmailMessage --- Messaging.SingleEmailMessage mail = new Messaging.SingleEmailMessage(); mail.setTargetObjectId(caseRecord.ContactId); mail.setSaveAsActivity(True); mail.setSubject('Update Regarding Your Case: ' + caseRecord.CaseNumber); String emailBody = 'Dear ' + (String.isNotBlank(caseRecord.Contact.FirstName) ? caseRecord.Contact.FirstName : 'Customer') + ',\n\n'; emailBody += 'This is a confirmation that we have processed your request related to Case ' + caseRecord.CaseNumber + '.\n'; emailBody += 'A follow-up will be conducted within the next few hours to ensure everything is resolved to your satisfaction.\n\n'; emailBody += 'Thank you for your patience.\n\n'; emailBody += 'Sincerely,\n'; emailBody += 'Support Team'; // Sender's name mail.setPlainTextBody(emailBody); mail.setWhatId(caseRecord.Id); Messaging.sendEmail(new Messaging.SingleEmailMessage[] {mail}); System.debug('Successfully sent email to Contact: ' + caseRecord.Contact.Email); Messaging.SendEmailResult[] results = Messaging.sendEmail(new Messaging.Email[] {mail}); if (results != null && results[0].isSuccess()) { System.debug('Email sent successfully test'); } else { System.debug('Email failed to send: ' + results[0].getErrors()[0].getMessage()); } // Set success response. response.isSuccess = true; response.outcomeMessage = 'Follow-up task created and email sent successfully for Case: ' + caseRecord.CaseNumber; } catch (Exception e) { response.isSuccess = false; response.outcomeMessage = 'An error occurred: ' + e.getMessage(); System.debug('CaseFollowUpAgentAction ~~ Exception Type: ' + e.getTypeName() + ', Message: ' + e.getMessage() + ', Stack Trace: ' + e.getStackTraceString()); } responses.add(response); return responses; } public class FollowUpRequest { @InvocableVariable(required=true label='Case Number' description='Stores the case number of the existing case for which a follow-up task needs to be created.') public String caseNumber; public FollowUpRequest() { } } public class FollowUpResponse { @InvocableVariable(label='Outcome Message' description='A message indicating the success or failure of the operation.') public String outcomeMessage; @InvocableVariable(label='Is Success' description='True if the operation was successful, false otherwise.') public Boolean isSuccess = false; } }
  • The @InvocableMethod annotation makes the createFollowUpTaskAndEmail method available to Agentforce and other Salesforce automation tools.
  • The `label` and `description` in `@InvocableMethod` are important as they help Agentforce understand how to use this action.
  • It takes a list of FollowUpRequest objects and returns a list of FollowUpResponse objects.

Step 2: Add Extra Instructions to Existing Topic

Enhance your existing Agentforce topic with new instructions to guide the agent on when to use the follow-up action.

New Instructions:

  • Instruction 1 (Task Definition):
    • "A Task record is used as a follow-up action for a Case. [cite: 35] It's assigned to the relevant team to highlight the issue and ensure a prompt resolution." [cite: 36]
  • Instruction 2 (Triggering Condition):
    • "If a customer indicates their issue on an existing case remains unresolved, you must first obtain the case number before creating a follow-up task record to prompt further action." [cite: 37]

Add these instructions to the topic you configured previously. This helps the agent understand the context and prerequisites for the new "Create Follow-Up Task" action.

Step 2: Create the Agent Custom Action Exposing Apex to Agent

Now, create an Agent Custom Action to make your Apex method usable by the Agentforce agent.

Action Configuration:

  • From Setup, open Agent Assets and click on the Actions Tab.
  • Click New Agent Action.
  • Configure the action as follows:
FieldValue
Reference Action TypeApex
Reference Action CategoryInvocable Methods
Reference ActionCreate Follow-Up Task & Email (label from @InvocableMethod)
Agent Action LabelKeep default (e.g., Create Follow-Up Task & Email)
Agent Action API NameKeep default
  • Click Next.

Notice that the instruction fields are pre-filled based on the descriptions provided in the Apex code's @InvocableMethod and @InvocableVariable annotations.

Step 3: Create Agent Custom Action Input/Output Settings

Configure the input and output behavior for the custom action.

Settings:

  • Set the Loading Text to: "Creating followup task for you...".
  • For the input parameter (e.g., "Case Number" from FollowUpRequest):
    • Check Collect data from user.
  • For the output parameter (e.g., "Outcome Message" from FollowUpResponse):
    • Check Show in conversation.
  • Click Finish.

Step 3: Add the Action to Your Agent

Integrate the newly created custom action into your agent's existing topic.

Steps:

  • From Setup, open Agentforce Agents.
  • Click your agent (e.g., Agentforce).
  • Click Open in Builder. [cite: 45]
  • In the Topics sidebar, click on the topic you created/updated previously.
  • Click the This Topic's Actions tab.
  • Click the New button, then Add from Asset Library.
  • Check the Create Follow-Up Task & Email action (or the label you used).
  • Click Finish.

Remember to re-activate your agent if it was deactivated for these changes.

Step 5: Try It Out! Testing the Apex Action

Test the new functionality using the Conversation Preview panel.

Scenario Prompt:

Enter the following prompt into the Conversation Preview panel:

I am following up on the issue I raised earlier. My air conditioning is still not working, and it's getting very hot in the room. Can you please look into this on priority and resolve it as soon as possible? [cite: 50]

Conclusion & Key Learnings

You have successfully created and integrated an Apex-based custom action into your Agentforce agent!

Key Takeaways:

  • Apex classes with @InvocableMethod can extend Agentforce capabilities significantly.
  • Input (FollowUpRequest) and output (FollowUpResponse) structures are defined using inner classes with @InvocableVariable.
  • Descriptions in annotations are crucial for Agentforce to understand and configure the action.
  • Agent Custom Actions bridge the gap between your Apex logic and the agent.
  • Topic instructions guide the agent on when and how to use custom actions.

Further Exploration:

  • Implement more complex Apex logic (e.g., callouts to external systems, advanced queries).
  • Create actions with multiple input or output parameters.
  • Explore different ways to handle errors and provide feedback to the user.
